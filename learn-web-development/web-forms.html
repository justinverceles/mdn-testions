Core forms learning pathway

How to structure a web form

  1. What element formally defines a form?

    <form></form>

    1. Can you use a form control outside of this element?

      Yes, but if you do so, that control has nothing to do with any form unless you associated it with a form using the form attribute whose value is the id of the <form></form>.

  2. What element groups together widgets which share the same purpose, for styling and semantic purposes?

    <fieldset></fieldset>

    1. How can you provide a label for this element?

      Between <legend></legend> tags beneath the opening <fieldset> tag

    2. How do screen readers and other assistive technologies use these elements?

      They'll speak the legend's content before speaking the label of each control.

  3. How do you label a widget?

    With a <label></label> element using a for attribute whose value is the id of the widget(s). The label and the widgets can be separate or the widgets can be nested in the label.

    1. What happens if you click or tap the label for a widget?

      It activates the corresponding widget.

    2. Should you use multiple labels for a single widget? Why or why not?

      No because some assistive technologies have difficulty handling them.

Basic native form controls

  1. Can you perform rich editing (bold, italic, etc.) on HTML form text fields? Why or why not?

    No, because they're all plain text input controls. All rich text editors you encounter are custom widgets created with HTML, CSS, and JavaScript.

  2. How do you make it so a user cannot modify a text input's value but it is still sent with the rest of the data?

    Mark it with a boolean readonly attribute.

  3. How do you give a text input placeholder text that briefly describes the purpose of the box?

    Use the placeholder attribute.

  4. How do you control the size of a text input box?

    Use the size attribute whose value is the number of characters wide the box is.

  5. How do you control the maximum amount of characters that can be entered into a text input?

    Use the maxlength attribute.

  6. How do you control whether a text input is checked for spelling errors?

    Use the spellcheck attribute, whose value you have to explicitly specify as true or false (it's not a boolean attribute).

  7. How do you make a single line text field?

    <input type="text"> or you could omit the type attribute because text is the default value.

  8. How does the browser deal with line breaks in single line text fields?

    It removes those line breaks before sending the data to the server.

  9. How do you make a password field?

    <input type="password">

    1. How is this different from a regular text input?

      It obscures the value entered with dots or asterisks.

    2. How secure is this?

      It's sent in plain text so it's not secure at all unless it's sent over a secure connection (HTTPS).

  10. How do you create a form control which is invisible to the user but is still sent to the server along with the rest of the data once submitted?

    <input type="hidden">

    1. What attributes are required with this input type?

      name and value

  11. How do you create a checkbox input?

    <input type="checkbox">

  12. How do you create a radio button input?

    <input type="radio">

  13. How do you make it so a checkbox or radio button is checked by default?

    Mark it with the boolean checked attribute.

  14. What determines whether a checkbox or radio button input is sent when the form is submitted?

    Their values are only sent if they are checked.

    1. What happens if you send one with no value?

      It is sent with a value of on.

  15. How do you make it so that, out of a group of radio buttons, only one can be checked at a time?

    Give them all the same name attribute value.

  16. How do you make a button that sends data to the server?

    <button type="submit">button</button> or you could omit the type attribute because submit is the default value. It also defaults to submit if you put an invalid value of type.

  17. How do you make a button that resets all form widgets to their default values?

    <button type="reset">button</button>

  18. How do you make a button that has no automatic effect but can be customized with JavaScript?

    <button type="button">button</button>

  19. How do you make an image button, which renders like an <img> but behaves like a submit button? What does this send to the server?

    <input type="image" src="some-image.png">

    This sends the X and Y coordinates of the click on the image, with the upper-left corner being the origin (0,0). The X-value key is the value of the name attribute followed by the string ".x". The Y-value key is the value of the name attribute followed by the string ".y".

  20. How do you make a file picker which accepts images?

    <input type="file" accept="image/*">

  21. How do you make a file picker which accepts images captured directly by the device's camera?

    <input type="file" accept="image/*;capture=camera">

  22. How do you make a file picker which accepts video captured directly by the device's camcorder?

    <input type="file" accept="video/*;capture=camcorder">

  23. How do you make a file picker which accepts audio captured directly by the device's microphone?

    <input type="file" accept="audio/*;capture=microphone">

The HTML5 input types

  1. How do you make an email address field?

    <input type="email">

    1. How do you make it so multiple email addresses can be entered into the same input?

      Use the boolean multiple attribute.

    2. How does this change the UI on some devices?

      Touch devices with dynamic keyboards will show a keypad with an @ symbol.

  2. Why should error validation be done on the server side when built-in client-side error validation exists?

    Client-side validation is easy to turn off, so malicious users can easily send bad data through to your server.

  3. How do you make a search field?

    <input type="search">

  4. How do you make a phone number field?

    <input type="tel">

    1. Why is this type useful even when you don't need the user to enter a phone number?

      Touch devices with dynamic keyboards will show a numeric keypad, which is useful for more than just phone numbers.

  5. How do you make a URL field?

    <input type="url">

    1. What kind of validation does this provide?

      The browser will report an error if no protocol was entered.

    2. How does this change the UI on some devices?

      Touch devices with dynamic keyboards will often display some or all of the colon, period, and forward slash as default keys.

  6. How do you make a numeric field?

    <input type="number">

    1. How does this change the UI on different devices?

      Looks like a text field but allows only floating-point numbers, and usually provides a spinner to increase and decrease the value. On touch devices with dynamic keyboards, a numeric keyboard is displayed.

    2. How do you constrain the minimum and maximum values?

      Use the min and max attributes.

    3. How do you set the increment and decrement caused by pressing the spinner buttons?

      Use the step attribute.

      1. What is the default value of this?

        1, meaning only whole numbers are valid

      2. How do you make it so it increments by one but allows floating point numbers?

        step="any"

  7. How do you make a slider control?

    <input type="range">

    1. How do you set the minimum, maximum, and increment values respectively for this?

      min, max, and step attributes

    2. How do you make it so the current value is displayed?

      Use an <output></output> element whose for attribute is the <input>'s id. Then, the JavaScript:

      const slider = document.querySelector(range input);
const output = document.querySelector(output element);
output.textContent = slider.value;
slider.addEventListener('input', function(){
  output.textContent = slider.value;
});

  8. How do you make a control to pick a date and time with no specific timezone information?

    <input type="datetime-local">

  9. How do you make a control to pick a month and a year?

    <input type="month">

  10. How do you make a control to pick a time value?

    <input type="time">

    1. How does the way this displays differ from what returns?

      Displays in a 12-hour format while the value returned is in 24-hour format

  11. How do you make a control to pick a week number and a year?

    <input type="week">

    1. How is the first week of the year determined?

      The first week of the year contains the first Thursday of the year, which may not include the first day of the year or may include the last few days of the previous year.

  12. How do you constrain the minimum, maximum, and increment values respectively for date/time values?

    min, max, and step attributes

  13. How do you make a color picker control?

    <input type="color">

Other form controls

  1. How do you make a multi-line text field?

    <textarea></textarea>

    1. How does the browser deal with hard line breaks in multi-line text fields?

      They are included when the data is submitted.

    2. How do you give a multi-line text field some default text?

      Put it in between the tags.

    3. How do you size a multi-line text field? What is its default size?

      Use the cols attribute which is the width measured in average character widths, and rows which is the height in number of text rows.

      The default size is cols="20" and rows="2"

    4. How do you control how a multi-line text field wraps text? What values does this take and what is the default value?

      Use the wrap attribute whose values are as follows:

      soft
      Rendered as wrapped but not submitted as wrapped
      hard
      Both rendered and submitted as wrapped
      off
      Stops wrapping

      The default value is soft.

    5. How do you use CSS to control the user's ability to resize a multi-line text field?

      Use the resize property whose values are as follows:

      both
      The default. Allows resizing horizontally and vertically
      horizontal
      Allows resizing only horizontally
      vertical
      Allows resizing only vertical
      none
      Allows no resizing
      block or inline
      Allows resizing only in the block or inline direction

  2. How do you make a select box whose options are "Banana", "Cherry", and "Lemon"?

    <select>
  <option>Banana</option>
  <option>Cherry</option>
  <option>Lemon</option>
</select>

    1. How do you set the default value of the select box?

      Use the boolean selected attribute on that option.

    2. How do you make a select box whose options are "Banana", "Cherry", "Lemon", "Carrot", "Eggplant", and "Potato", and the options are grouped into "fruits" and "vegetables"?

      <select>
  <optgroup label="fruits">
    <option>Banana</option>
    <option>Cherry</option>
    <option>Lemon</option>
  </optgroup>
  <optgroup label="vegetables">
    <option>Carrot</option>
    <option>Eggplant</option>
    <option>Potato</option>
  </optgroup>
</select>

    3. What determines which value is sent when the form is submitted?

      You can explicitly set the values on the <option></option>s, but if you omit them, then the values sent will just be the content of the <option></option>s.

    4. How do you control how many options are visible when the select does not have focus?

      Use the size attribute.

    5. How do you make it so a user can select multiple options? How does this change the UI?

      Use the boolean multiple attribute on the <select></select> so users can select multiple options by holding down Cmd/Ctrl. If you do this, the values no longer display as drop-down content; instead they display in a list, with an optional size attribute determining the height.

  3. How do you provide a form widget with suggested, autocompleted values?

    Use the <datalist></datalist> element with child <option></option>s whose id value is the related control's list value.

    1. How can you provide fallback for this to support older browsers like IE versions below 10?

      <datalist id="the-datalist">
  <select>
    <option>option 1</option>
    <option>option 2</option>
    <option>option 3</option>
  </select>
</datalist>

  4. How do you give a slider tick marks above specific values?

    Associate a <datalist></datalist> with the slider. Each child <option></option> and its value is a tick mark.

  5. How do you give a color picker a customized palette of colors, while still making the full color palette available?

    Associate a <datalist></datalist> with the color picker.

  6. How do you make a progress bar with fallback?

    Use the <progress></progress> whose max value is the maximum value and whose value is the value. The fallback would be between the tags like "75/100".

  7. How do you make a meter bar with fallback?

    Use the <meter></meter> whose range is divided like so:

    • The lower part of the range is between the min and low values, inclusive
    • The medium part of the range is between the low and high values, exclusive
    • The higher part of the range is between the high and max values, inclusive
    • The optimum attribute defines the optimum value, to which the value is compared to determine how to color the bar

    The fallback works the same as with <progress></progress>.

Advanced form styling

  1. Why are form controls often so difficult to style?

    Historically, styling of web form controls was largely taken from the underlying operating system.

  2. How do you stop a control from using system-level styling so that you can build up the styles yourself using CSS?

    Set appearance: none and -webkit-appearance: none

  3. What should you remember about prefixed properties? Why?

    You should include both the prefixed and unprefixed versions because "prefixed" usually means "work in progress" so in the future browser vendors may come to a consensus to drop the prefix.

UI pseudo-classes

  1. What pseudo-classes target elements that are being hovered over, focused, and activated respectively?

    :hover, :focus, and :active

  2. How do you make it so a user has to fill in a certain control in order for the form to successfully submit? How do you target these controls using CSS?

    Mark it with the boolean required attribute. These are targeted by the :required pseudo-class.

  3. How do you make it so a user does not have to fill in a certain control in order for the form to successfully submit? How do you target these controls with CSS?

    Don't mark it with the boolean required attribute. These are targeted by the :optional pseudo-class.

  4. How do you style controls based on whether their data is valid?

    :valid and :invalid

  5. How do you target controls whose current value is outside the range set by the min and max attributes?

    :invalid or :out-of-range

    1. What if it is inside that range?

      :in-range

  6. How do you target controls with disabled set on them?

    :disabled

    1. What about controls without disabled set on them?

      :enabled

  7. How do you target controls that have their values submitted to the server but the user can't edit them?

    :read-only (targets elements with the boolean readonly attribute)

    1. What about controls that can be edited?

      read-write

  8. How do you target radio buttons and checkboxes that have the checked attribute?

    :default

    1. What if the user unchecks them?

      :default still applies

  9. How do you target radio buttons and checkboxes that are checked?

    :checked

  10. How do you target radios/checkboxes that are neither checked nor unchecked? When would this be the case?

    :indeterminate which applies for:

    • <input type="radio">s when all radio buttons in a same-named group are unchecked
    • <input type="checkbox" whose indeterminate property is set to true via JavaScript
    • <progress></progress>s that have no value

  11. How do you target elements that have received focus or contain elements that have received focus?

    :focus-within

  12. How do you target focused elements that have received focus via keyboard interaction (rather than touch or mouse)?

    :focus-visible

  13. How do you target <input>s and <textarea></textarea>s that have their placeholder text showing because the value of the element is empty?

    :placeholder-shown

Client-side form validation

  1. How does built-in form validation differ from form validation done with JavaScript?

    Built-in form validation has better performance, but isn't as customizable as JavaScript validation.

  2. How do you specify the minimum and maximum length of textual data?

    Use the minlength and maxlength attributes.

  3. How do you define the pattern that textual data needs to follow?

    Use the pattern attribute, which specifies a "regular expression".

  4. What JavaScript API is used for form validation?

    Constraint Validation API

    1. What property returns a localized message describing the validation constraints that the control doesn't satisfy (if any)? In what scenarios would this return an empty string?

      validationMessage

    2. What property returns a ValidityState object containing several properties describing the validity of the element?

      validity

      1. What property returns true if the value does not match the specified pattern, and false if it does match?

        patternMismatch

      2. What property returns true if the value is longer than the specified maxlength value, and false if it's shorter or equal?

        tooLong

      3. What property returns true if the value is shorter than the specified minlength value, and false if it's greater than or equal?

        tooShort

      4. What property returns true if the value is greater than the specified max value, and false if it's less than or equal?

        rangeOverflow

      5. What property returns true if the value is less than the specified min value, and false if it's greater than or equal?

        rangeUnderflow

      6. What property returns true if the value is not in the required syntax (when type is email or url), and false if the syntax is correct?

        typeMismatch

      7. What property returns true if the value is valid and false if it's invalid?

        valid

      8. What property returns true if the element has a required attribute, but no value, and false if otherwise?

        valueMissing

    3. What property returns true if the element will be validated when the form is submitted, and false if otherwise?

      willValidate

    4. What method returns true if the element has no validity problems, and false if otherwise?

      checkValidity()

    5. What method adds a custom error message (in the parameters) to the elements and, if set, renders the element invalid?

      setCustomValidity(the message)

Sending form data

  1. How do you define where a form's data gets sent? What happens if you omit this?

    Use the action attribute whose value is a valid absolute or relative URL. If omitted, the data is sent to the URL of the page containing the form.

    1. What happens if you specify an HTTP URL as opposed to an HTTPS one? What if the form is hosted on an HTTPS page?

      All browsers will display a security warning to the user each time they try to send the data because the data will not be encrypted. It will do this regardless of whether the form is hosted on an HTTP page or an HTTPS page.

  2. What does an HTTP request consist of?

    A header that contains a set of global metadata about the browser's capabilities, and a body that contains information necessary for the server to process the specific request.

  3. How do you define how a form's data is sent?

    Use the method attribute.

    1. What are the two main values this takes? Describe them.

      GET:

      • Asks the server to send back a given resource
      • HTTP request body is empty, data sent to the server is appended to the URL
      • e.g. if action="https://www.example.com", https://www.example.com/?name=value&name2=value2

      POST:

      • Asks the server for a response that takes into account the data provided in the body of the HTTP request
      • data appended to the HTTP request body, not the URL

  4. What languages/frameworks are generally used for retrieving data on the server side?

    PHP, Python, Django for Python, Express for Node.js, Laravel for PHP, Ruby on Rails for Ruby

  5. How do you send files on the client-side? Why is this necessary?

    1. Set method to POST because file content can't be put in URL parameters
    2. Set the value of the enctype attribute (whose default value is application/x-www-form-urlencoded) to multipart/form-data because the data will be split into multiple parts, one for each file plus one for the text data included in the form body (if text is also entered into the form)

    This is necessary because files are binary data and HTTP is a text protocol.
