  17. What is the event object?

Asynchronous JavaScript

General asynchronous programming concepts

  1. What does asynchronous programming do?

    It allows you to run other tasks while you're waiting for a task to complete.

  2. What is blocking?

    This is when, while code is being executed, the browser is kept from continuing to handle user input or performing other tasks until complete.

  3. What is a thread? Why is JavaScript called single-threaded?

    A thread is a single sequence of tasks, which is run one by one. JavaScript can traditionally only handle one thread at a time, called the main thread.

  4. How do web workers work?

    Web workers will run other tasks in a separate thread called a worker thread while the tasks in the main thread continue to run, so user interaction is not blocked.

  5. What is one limitation to web workers?

    Web workers aren't able to access the DOM, so you can't get a worker to directly do anything to update the UI.

Callbacks

  1. What are async callbacks?

    Functions that are specified as arguments when calling a function which will start executing in the background. When the background code finishes running, it calls the callback function to let you know the work is done, or to let you know that something of interest has happened.

Timeouts and intervals

  1. What function executes a specified block of code once after a specified time has elapsed?

    setTimeout()

    1. What parameters does this take?

      1. A function to run
      2. A number representing the time interval in milliseconds
      3. Zero or more values that represent any parameters you want to pass to the function when it is run.

    2. Is the specified time the guaranteed time to execution or the minimum time? Why?

      The minimum time. This is because the callbacks you pass to these functions cannot run until the stack on the main thread is empty. If you execute code like setTimeout(fn, 0) but then immediately after run a loop that counts from 1 to 10 billion, your callback will be executed after a few seconds.

    3. Do callback functions passed to this function have to be anonymous?

      No, you can have something like setTimeout(function sayHi(){code}, 2000)

  1. What does API stand for?

  2. What are APIs?

    APIs are constructs made available in programming languages to allow developers to create complex functionality more easily. They abstract more complex code away from you, providing some easier syntax to use in its place.

  5. What are JavaScript frameworks?

    The next step up from libraries. Tend to be packages of HTML, CSS, JavaScript, and other technologies that you install and then use to write an entire web application from scratch.

    1. What are service workers?
