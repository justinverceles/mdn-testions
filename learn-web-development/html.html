Introduction to HTML

Getting started with HTML

  1. What does HTML stand for?

    Hypertext Markup Language

  2. Label the three parts of the following HTML element:

    <p>My cat is very grumpy</p>

    1. How would you put the word "very" in bold using HTML?

      <p>My cat is <strong>very</strong> grumpy</p>

  3. What are the differences between block-level elements and inline elements?

    A block-level element appears on a separate line from the preceding and following content. It wouldn’t appear nested within an inline element but it might within another block element. Block-level elements are usually structural elements on the page.

    An inline element is contained within a block-level element, and does not appear on a separate line from other content. Inline elements are typically associated with text.

  4. What do you call an element that consists of a single tag?

    An empty element or a void element

  5. How might you add a class of editor-note to the element from Question 2? What is this sort of construction called?

    <p class="editor-note">My cat is very grumpy</p>

    This is called an attribute (with editor-note being its value).

  6. How would you code a link Go to Google which links to Google, shows a tooltip reading "Google" when hovered over, and opens in a new tab?

    <a href="http://google.com" title="Google" target="_blank">Go to Google</a>

  7. How do you code a disabled text input? What do you call this kind of attribute?

    <input type="text" disabled="disabled">
    or 
    <input type="text" disabled>

    This is called a boolean attribute.

  8. What is the purpose of <!DOCTYPE html> at the top of every HTML document?

    Used to act as a link to a set of rules the HTML had to follow to be considered good HTML. Now it’s just a historical artifact that needs to be included for everything to work right. This is the shortest string of characters that counts as a valid doctype.

  9. How do you specify the character set for your document so that it includes most characters from the vast majority of human written languages?

    <meta charset="utf-8"> in the <head></head>

  10. How do you give a webpage a title to be displayed in the tab?

    Put it in between <title></title> tags in the <head></head>.

  11. How does the HTML parser deal with whitespace?

    The HTML parser reduces all whitespace down to a single space.

  12. How would you write the following characters without them being mistaken for HTML? What are these called?

    • <
    • >
    • "
    • '
    • &

    These are called character references.

    • &lt;
    • &gt;
    • &quot;
    • &apos;
    • &amp;

  13. How do you write comments in HTML?

    <!-- This is a comment -->

The head

  1. How would you code metadata about the author of a page and a description of the page, into the head? Why is this useful to do?

    <meta name="author" content="Justin Verceles">
<meta name="description" content="This is a website">

    It can be useful to have this information. Content management systems (CMS) have facilities to automatically extract this sort of information. Keywords in the description also aid in Search Engine Optimization (SEO).

  2. Searching for "MDN Web Docs" on Google shows relevant subpages below the main homepage link. What are these called and how can you configure them for your own website?

    These are called sitelinks. They can be configured using the Google Search Console.

  3. What are Twitter and Facebook's respective properietary metadata protocols?

    Facebook uses Open Graph Data. Twitter uses Twitter Cards.

  4. What do you call the custom icon that displays next to a title in a browser tab and in a bookmark? How do you add this to your site?

    <link rel="shortcut icon" href="favicon.ico" type="image/x-icon">

    1. In what format should you save this icon? Why?

      Browsers will support favicons in formats like .png or .gif but saving it as an .ico will ensure it works as far back as IE6.

  5. How do you add CSS and JavaScript to your HTML through the head?

    <link rel="stylesheet" href="styles.css">
<script src="script.js" defer></script>

  6. How do you set the primary language of the document to US English? What about of a single inline element to Japanese?

    Use the lang attribute e.g.

    <html lang="en-US"></html>
    or 
    <span lang="ja"></span>

    1. What are these codes called?

      ISO 639-1 to specify language and ISO 3166-1 to specify country.

HTML text fundamentals

  1. What element is used to designate a paragraph?

    <p></p>

  2. What elements are used to designate headings?

    (closing tags omitted for brevity) <h1>, <h2>, <h3>, <h4>, <h5>, and <h6>

  3. What's the maximum number of heading levels you should use per page?

    three

  4. How many top-level headings should you use per page?

    one

  5. Why should you write semantic markup?

    Structure makes it easier for people to find what they’re looking for if they’re just scanning the page. Makes it easier for visually impaired users to use your website with their screen readers. Headings are good for SEO because search engines consider headings important keywords. Makes it easier to target elements with CSS and Javascript.

  6. How do you write an unordered list?

    <ul>
  <li>milk</li>
  <li>eggs</li>
<ul>

  7. How do you write an ordered list?

    <ol>
  <li>Make a left on Main St.<li>
  <li>Turn right on Primary Blvd.<li>
<ol>

  8. Can you nest lists?

    Yes.

  9. What are the differences between emphasis and strong importance? What elements are used for them?

    Emphasis (<em></em>) changes the meaning of the sentence and is styled by default with italics.

    Strong importance (<strong></strong>) doesn't change the meaning — it just stresses a certain word. It is styled by default as bold text.

  10. What are the elements used for bold, underline and italics? What are the more recently defined semantic roles for these elements?

    <b></b>, <u></u>, and <i></i>. These elements should be used to convey meanings that are traditionally used to convey, respectively:

    • Keywords, product names, lead sentences…
    • Proper names, misspellings…
    • Foreign words, taxonomic designations, technical terms, thoughts…

  11. What do you call elements that don't affect semantics? Should they be used?

    Presentational elements. They shouldn't be used. It's better to use CSS for presentation.

  12. What is problematic about the underline element and how is this solved?

    Underlines are strongly associated with links. You can use CSS to make the underline look less like a link.

Creating hyperlinks

  1. What is important to remember about the title attribute?

    Its content is largely inaccessible to users who rely on screen readers, keyboard controls, or touchscreens.

  2. How do you make a block-level link?

    Enclose the block-level element in <a></a> tags.

  3. How do you link to something in the same directory as the HTML file?

    Just the filename.

  4. How do you link to something in a subdirectory from where the HTML file is?

    Use /

  5. How do you link to something in a parent directory from where the HTML file is?

    Use ../

  6. What do you call one specific part of an HTML document? How do you link to one?

    Document fragment. Let's say you're linking to a specific heading in the document. You would give that heading an id attribute, then link to the heading using a # followed by the id value. e.g.

    <a href="#chapter-one">Chapter One</a>

  7. What does URL stand for?

    Uniform Resource Locator

  8. What is the difference between an absolute URL and a relative URL?

    An absolute URL points to an absolute location on the web, including protocol and domain name. A relative URL points relative to the file you're linking from.

  9. Is it better to use absolute URLs or relative URLs?

    Relative URLs because using an absolute URL requires the browser to look up the location on the DNS (Domain Name System). Relative URLs also tend to be shorter and easier to read when scanning code.

  10. What are some best practices to follow when writing link text?

    Don't repeat link text. Don't use phrases like "links to" because users can tell it's a link either by looking at it or their screen reader tells them. Don't use the URL in the link text because they look ugly and a screen reader will read it aloud letter-for-letter.

  11. What are some best practices to follow when writing links that point to non-HTML resources?

    Indicate in the link text that it's a non-HTML resource. If it's a download, indicate the size of the file and use the download attribute to give the file a default name.

  12. How would you write a link that sets up an email to lentils@gmail.com and legumes@gmail.com, a bcc to alfalfa@gmail.com, with a subject, "Regarding the letter you sent me", and a body, "Very nice penmanship"? How do you include those spaces in the link and what is that called?

    mailto:lentils@gmail.com?cc=legumes@gmail.com&bcc=alfalfa@gmail.com&subject=Regarding%20the%20letter%20you%20sent%20me&body=Very%20nice%20penmanship
    or 
    mailto:lentils@gmail.com,legumes@gmail.com?bcc=alfalfa@gmail.com&subject=Regarding%20the%20letter%20you%20sent%20me&body=Very%20nice%20penmanship

    The %20s are an example of percent encoding.

  13. How much is optional in an email link?

    Everything is optional except the mailto: which just opens up your email.

Document and website structure

  1. What are the basic sections of a webpage? What tags denote them?

    header
    <header></header>
    navigation bar
    <nav></nav>
    main content
    <main></main>
    sidebar
    <aside></aside>
    footer
    <footer></footer>

  2. Where does the navigation bar go in relation to the header?

    It could be part of the header, or separate from the header which might be better for accessibility (screen readers can read them better)

  3. What is the purpose of the sidebar?

    For information directly or indirectly related to the main content. Could be used as a secondary navigation system.

  4. What is the purpose of the footer?

    For common information (like in the header) but not critical for the website itself. Can also contain links for quick access to popular content.

  5. How is main content subdivided?

    into <section></section>s and <article></article>s. Articles are for grouping related content that makes sense on its own without the rest of the page. Sections are for grouping related content that designates one piece of functionality. Sections can be divided into articles or the other way around.

  6. How is <header></header> used?

    If it's a child of <body></body>, it's the global header for the website. If it's a child of <section></section> or <article></article>, it's a specific header for that section (contains an h1-h6 and possibly information about the author and the date the article was published).

  7. What are the non-semantic wrappers?

    <div></div>s for block-level, <span></span>s for inline

  8. How do you code a line break?

    <br>

  9. How do you code a horizontal rule?

    <hr>

  10. What is information architecture?

    Working out what content you want to put on a whole website, what pages you need, and how they should be arranged and link to one another for good UX

  11. What are the steps to planning a simple website?

    1. Note what content should be common to every page.
    2. Sketch what the structure of each page should look like.
    3. Brainstorm all other content (not common to each page) you want to have on your website.
    4. Try to sort all these content items into groups, to get an idea of what content should be together on different pages.
    5. Sketch a sitemap, each page being a bubble with the content they will contain, and arrows showing how the pages will be related.

Debugging HTML

  1. What are the two main types of errors in code?

    Syntax errors (basically spelling errors) and logic errors (code doesn't work as intended)

    1. Which of these does HTML not suffer from as much? Why?

      Browsers parse HTML permissively, meaning it will correct syntax errors.

Multimedia and embedding

Images in HTML

  1. How do you embed an image dinosaur.png into a page?

    <img src="dinosaur.png">

  2. What should you always make sure of first before you try to display an image on your page?

    You own the image; you have written explicit permission from the owner; you have ample proof that the image is in the public domain

  3. What is hotlinking? Why shouldn't you do it?

    Hotlinking means pointing your src attribute to an image on someone else's website. This is stealing bandwidth and is illegal. You also have no control over whether the image is changed or removed.

  4. What do you call elements like the image element or the video element whose content and size is defined by an external resource?

    Replaced elements

  5. Why should you use alternative text? How do you code it?

    For when the image doesn't load, for visually impaired people, and to provide text for search engines to utilize. To code it, you use the alt attribute.

  6. What should you write in alternative text?

    If the image provides significant information, the alt text should provide that same information, unless that information is already in the main text. If the image is in <a></a>s, put accessible link text.

  7. How do you make an image load smoother using width and height attributes?

    Provide width and height attributes with the same dimensions as the image, that way the space is already loaded before the image loads.

  8. How do you give a piece of content a caption?

    Put the content in <figure></figure> tags, then put the <figcaption></figcaption> tags containing the caption inside the <figure></figure> tags.

iframes

  1. How might you embed a YouTube video into a page?

    You can do this by clicking the "Share" button under a YouTube video then clicking "Embed" and copying the code.

  2. What are the security concerns with using iframes?

    iframes are a common attack vector for hackers to maliciously modify your webpage or trick people into revealing sensitive information. Clickjacking is when a hacker embeds an invisible iframe in your webpage (or embed your document into their malicious website) to capture users' interactions, misleading users or stealing sensitive data.

  3. How can you safely use iframes?

    Only embed when absolutely necessary. If you made it, double-check it anyways. If someone else made it, assume it's dangerous until proven otherwise.

    Use HTTPS, which reduces the chance that remote content was tampered with in transit, and prevents embedded content from accessing your content in your parent document and vice versa. HTTPS requires a security certificate which can be expensive, but never embed third-party content with HTTP. At best, users will get a scary warning from this.

    Always use the sandbox attribute.

  4. How can you give an attribute multiple values?

    Separate them with spaces in the quotes e.g. class="val1 val2 val3"

  5. What two values should never both be added to the sandbox attribute? Why?

    allow-scripts and allow-same-origin. This allows embedded content to bypass the same-origin policy which prevents it from executing scripts, and then use JavaScript to turn off sandbox altogether.

  6. How do you prevent other websites from embedding your content in their webpages?

    Content Security Policy (CSP) provides a set of HTTP headers (metadata sent along with your webpages when they're served from a server). You can configure your web server to send an appropriate X-Frame-Options header to prevent other websites from embedding your content.

Vector graphics

  1. What's the difference between a raster image and a vector image?

    A raster image is defined by a grid of pixels whereas vector images use algorithms to make shapes.

  2. What is SVG?

    Stands for Scalable Vector Graphics. It's an XML-based markup language for describing vector images.

  3. What are some advantages of SVG?

    • SVG doesn't get pixelated when you zoom in.
    • Text in an SVG image remains accessible.
    • SVG lends itself well to styling and scripting, because each component of the image is an element.

  4. What are some disadvantages of SVG?

    • SVG file sizes can get big quickly.
    • It's more difficult to create complex images with SVG.
    • Older browsers don't support SVG.

  5. How can you add SVG using <img>? What are the pros and cons of this method?

    Include the path to the SVG file in the src.

    Pros:

    • The browser caches the image so it loads faster after the first time you use it.
    • The syntax is simple and familiar
    • You can use alt text.

    Cons:

    • You can't style or script the components.
    • You can't style it without using inline CSS in the SVG file.

  6. How can you make SVG in an image responsive to browsers that don't support SVG?

    Put a raster version in src and the SVG version in srcset. Older browsers won't recognize srcset and will load the raster image.

  7. What do you call it when you put SVG code directly in your HTML? What are the pros and cons of doing this?

    That's called inline SVG, or inlining SVG.

    Pros:

    • The components can be scripted and styled.
    • Saves an HTTP request.

    Cons:

    • The browser can't cache it.
    • Duplicating the image can be resource-intensive.
    • It makes the file bigger.
    • You can include fallback using <foreignobject></foreignobject> in the SVG with display="none", but browsers that support SVG will still download any fallback images.

  8. How is the origin of web content defined?

    The scheme (protocol), host (domain), and port.

  9. What are the pros and cons of embedding SVG in an iframe?

    There are no pros.

    Cons:

    • You can't script it with JavaScript unless your webpage and the SVG have the same origin.
    • iframes have a fallback mechanism, but it only works if browsers lack support for iframes altogether.

Responsive images

  1. What is the difference between an art direction problem and a resolution switching problem?

    An art direction problem involves wanting to change the image to suit different display sizes. A resolution switching problem involves wanting to change the size or resolution of the image.

  2. What are the two kinds of resolution switching problems?

    • Different sizes
    • Same size, different resolution

  3. You have an image dinosaur.jpg with intrinsic dimensions 600 by 300 pixels, and an alternate image dinosaur-alt.jpg with intrinsic dimensions 300 by 150 pixels. You want dinosaur-alt.jpg to display when the width of the screen is 600 pixels or less, and dinosaur.jpg any other time.

    How do you code this to happen? If the browser doesn't support these functionalities, how do you make dinosaur.jpg the fallback? What kind of problem is this?

    This is a resolution switching problem, specifically one where you want the image to change size.

    <img
  srcset="dinosaur.jpg 600w,
          dinosaur-alt.jpg 300w"
  sizes="(max-width: 600px) 300px,
          600px"
  src="dinosaur.jpg"
  alt="a dinosaur">

    1. What unit can you not use for slot width?

      percentages (%)

    2. How does the browser figure this out?

      First, it looks at the width of the screen. Then, it finds the first media condition in the sizes list to be true. Then, it looks at the slot size associated with that media condition. Finally, it loads the image in srcset with a size equal to the slot size or, if there isn't one, the first image bigger than the chosen slot size.

    3. What if you wanted dinosaur.jpgto display only when the resolution of the screen is 2 device pixels per CSS pixel, displaying the same picture as fallback? What kind of problem is this?

      This is a resolution switching problem, specifically one where you want the image to change resolution.

      <img
  srcset="dinosaur-alt.jpg,
          dinosaur.jpg 2x"
  src="dinosaur.jpg"
  alt="a dinosaur">

    4. What if you wanted dinosaur.jpg to display only when the width of the screen is 600px or more, and a cropped version dionsaur-cropped.jpg to display only when the width of the screen is 599px or less, displaying dinosaur.jpg as fallback? What kind of problem is this?

      This is an art direction problem.

      <picture>
  <source media="min-width: 600px" srcset="dinosaur.jpg">
  <source media="max-width: 599px" srcset="dinosaur-cropped.jpg">
  <img src="dinosaur.jpg" alt="a dinosaur">
</picture>

    5. Why can't any of this be done with CSS or JavaScript?

      The browser preloads all the images before the main parser has time to load and interpret the CSS and JavaScript, to cut loading time.

    6. What if you wanted the browser to be able to choose between a version of the dinosaur image in .jpg vs. .webp?

      <picture>
  <source type="image/jpg" srcset="dinosaur.jpg">
  <source type="image/webp" srcset="dinosaur.webp">
  <img src="dinosaur.jpg" alt="a dinosuar">
</picture>

  4. What line do you put in the head of the document to force mobile browsers to tell the truth about their viewport width?

    <meta name="viewport" content="width=device-width"

HTML table advanced features and accessibility

  1. How do you put a caption on a table? Where does this go in the markup?

    Use the <caption></caption> element, with the content of the caption in between the tags. It should be the first child of <table></table>.

  2. What elements can you use to give more structure to tables? What are the advantages of doing this?

    <thead></thead>, <tbody></body>, and <tfoot></tfoot>. These don't actually do anything for accessibility, as screen readers don't do anything with them, but they make it easier to style parts of the table.

  3. What is the purpose of a table footer? Where does this go in the markup?

    It could be used as a "sum" or "total" row. It can go just under the opening <table> tag or just before the closing </table> tag. Either way, it renders just below the table.

  4. Where does the table header element go in the markup?

    Just under the opening <table> tag, or just under the <colgroup></colgroup> and <col>s if you have them.

  5. Should you nest tables?

    No, not if you don't absolutely have to. It makes for confusing markup, and is less accessible to screen readers.

  6. How could you indicate to a screen reader how a header relates to some table data?

    You could put scope attributes on the <th></th>s (values: col, row, colgroup, rowgroup). Or, you could put id attributes on the <th></th>s and then give the associated <td></td>s headers attributes with those ids as values (can have muliple values, like headers="location haircut" if those are the two associated headers).
